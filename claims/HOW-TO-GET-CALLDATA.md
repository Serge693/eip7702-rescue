# Как получить calldata для клейма

> Это главный документ. Прочитай его полностью перед тем как что-то делать.

---

## Что такое calldata и зачем она нужна

Когда ты нажимаешь кнопку "Claim" на сайте аирдропа — браузер формирует транзакцию
и отправляет её в твой кошелёк (MetaMask и т.д.) для подписи.

Эта транзакция содержит два ключевых поля:
- `to` — адрес контракта клейма (это и есть `contract` в нашем JSON)
- `data` — зашифрованный набор байт с инструкцией что делать (это `data` в JSON)

Наш инструмент делает то же самое, только атомарно: claim + sweep в одной транзакции.
Нам нужно достать эти поля ДО того как подписывать что-либо MetaMask'ом.

**Главное правило: НИКОГДА не подписывай транзакцию клейма в MetaMask на скомпрометированном кошельке.**
Это даст мошеннику время среагировать. Мы только смотрим, не подписываем.

---

## Способ 1 — DevTools (самый простой, работает всегда)

### Шаг 1: Открой сайт клейма в браузере
Зайди на страницу клейма (например, `claim.hedgey.finance` или сайт проекта).

### Шаг 2: Открой DevTools
Нажми **F12** (или правая кнопка мыши → "Просмотр кода").
Перейди на вкладку **Network** (Сеть).

### Шаг 3: Подключи скомпрометированный кошелёк
Подключи кошелёк к сайту. **Только подключи — ничего не подписывай.**

### Шаг 4: Нажми Claim
Нажми кнопку Claim на сайте. MetaMask откроется с запросом подписи.

**НЕ НАЖИМАЙ ПОДТВЕРДИТЬ В METAMASK — просто посмотри данные.**

### Шаг 5: Найди транзакцию в MetaMask
В окне MetaMask где просят подтвердить транзакцию:
1. Нажми на "HEX" (или "DATA") чтобы увидеть сырые данные
2. Скопируй поле **"To"** — это твой `contract`
3. Скопируй поле **"Data"** (длинная строка начинающаяся с 0x) — это твой `data`

### Шаг 6: Отмени в MetaMask
Нажми **Отклонить** в MetaMask. Транзакция не отправлена.

### Шаг 7: Заполни JSON файл
```json
{
  "name": "Название клейма",
  "network": "base",
  "contract": "0x...адрес из поля To...",
  "data": "0x...данные из поля Data...",
  "hint_tokens": []
}
```

## Способ 2 — Посмотреть транзакцию другого клеймера

Этот способ работает если аирдроп уже идёт и другие люди уже клеймили.

### Шаг 1: Найди контракт клейма
Обычно его публикуют в официальном Discord/Twitter проекта.
Или найди по имени на Basescan: https://basescan.org

### Шаг 2: Открой контракт на Basescan
Зайди на `basescan.org/address/АДРЕС_КОНТРАКТА`

### Шаг 3: Посмотри последние транзакции
На вкладке **Transactions** найди любую транзакцию с методом **"Claim Tokens"** или похожим.

### Шаг 4: Открой транзакцию
Кликни на хэш транзакции (начинается с 0x...).

### Шаг 5: Найди Input Data
Прокрути вниз до раздела **"Input Data"** или **"More Details"**.
Там будет кнопка **"Decode Input Data"** — нажми её.

Ты увидишь раскодированные параметры:
- `campaignId` — ID кампании
- `proof` — список хешей для твоего адреса (у каждого **свой**, нельзя копировать чужой!)
- `claimAmount` — сумма в wei

**Важно: `proof` у каждого кошелька свой!** Нельзя просто скопировать чужую транзакцию.

### Шаг 6: Получи свой proof
Большинство проектов предоставляет API для получения proof:
```
https://api.проект.com/claim?address=ТВОй_АДРЕС
```
Или он есть на странице клейма при подключении кошелька.

## Способ 3 — Для Hedgey (специфика этого контракта)

Hedgey — это платформа для вестинга. Функция клейма называется `claimTokens`.

### Что тебе нужно знать заранее:
1. **campaignId** — ID кампании, 16 байт (bytes16). Проект публикует его при анонсе.
2. **proof** — твой персональный Merkle proof. Берётся с API или страницы клейма.
3. **claimAmount** — сумма в wei (умножь количество токенов на 10^18 для стандартных токенов).

### Как выглядит calldata Hedgey:
```
Функция: claimTokens(bytes16 campaignId, bytes32[] proof, uint256 claimAmount)
Селектор: 0xd6eecb00
```

Пример готовой `data` строки:
```
0xd6eecb00
[campaignId — 32 байта где первые 16 нули, последние 16 — ID]
[offset массива proof]
[claimAmount]
[длина массива proof]
[элемент proof 1]
[элемент proof 2]
...
```

### Самый простой способ для Hedgey:
1. Зайди на страницу клейма Hedgey
2. Подключи кошелёк (только смотришь, не подписываешь)
3. Нажми Claim — MetaMask покажет данные
4. Скопируй `To` и `Data` из MetaMask (способ 1 выше)

## Способ 4 — Через DevTools Network tab (продвинутый)

Когда сайт делает запрос к своему бэкенду чтобы получить данные для клейма,
это видно во вкладке Network DevTools.

### Как:
1. Открой DevTools → Network
2. В фильтре напиши: `claim` или `proof` или `merkle`
3. Подключи кошелёк на сайте
4. Посмотри запросы — обычно один из них содержит JSON ответ с `proof`, `amount`, `campaignId`

Пример ответа который ищешь:
```json
{
  "proof": ["0xabc...", "0xdef...", ...],
  "amount": "1000000000000000000000",
  "campaignId": "0x..."
}
```

## Особые случаи

### Hedgey с вестингом (locked/vesting)

**Важно:** Если в кампании Hedgey тип `tokenLockup = Locked` или `Vesting`,
то после клейма ты получишь **не токены, а NFT** (план вестинга).

Что это значит:
- Rescue всё равно нужно делать — без него мошенник заберёт этот NFT
- NFT потом постепенно разлокирует токены
- Каждый разлок нужно будет клеймить отдельно

Как определить тип:
- На странице клейма написано "Vesting" или "Locked" → это NFT
- Написано "Unlocked" или просто сумма без оговорок → это сразу токены

### Plume (пример того что НЕ нужно делать)

В истории с Plume мы потеряли токены потому что:
1. Клейм и sweep были в разных транзакциях (не атомарно)
2. Пришёл WPLUME (wrapped), а мы ждали PLUME

Как бы мы сделали правильно с этим инструментом:
```json
{
  "name": "Plume Hedgey Claim",
  "network": "plume",
  "contract": "0x...hedgey на plume...",
  "data": "0x...calldata клейма...",
  "hint_tokens": [
    "0x...адрес PLUME...",
    "0x...адрес WPLUME...",
    "0x...адрес pUSD..."
  ]
}
```
`hint_tokens` перечисляет ВСЕ токены которые могут прийти — и PLUME, и WPLUME.
Тогда sweep забрал бы оба.

## Быстрая памятка

```
Что нужно узнать перед клеймом:
├── contract  = адрес контракта клейма (поле "To" в MetaMask)
├── data      = calldata (поле "Data" в MetaMask, начинается с 0x...)
└── hint_tokens = адреса токенов которые придут (спроси в Discord проекта)

Как не облажаться:
├── НЕ подписывай транзакцию в MetaMask — только смотри
├── Proof у каждого кошелька СВОЙ — не копируй у других
├── Сначала запусти с --dry-run чтобы проверить
└── hint_tokens добавляй все возможные варианты (wrapped, unwrapped, оба)
```

## Шаблон для быстрого заполнения

Скопируй в `claims/мой-клейм.json` и замени значения:

```json
{
  "name": "Название проекта — тип клейма",
  "network": "base",
  "contract": "0x_АДРЕС_КОНТРАКТА",
  "data": "0x_CALLDATA",
  "hint_tokens": [
    "0x_АДРЕС_ТОКЕНА_1",
    "0x_АДРЕС_ТОКЕНА_2"
  ]
}
```

Потом:
```bash
# Сначала проверь без отправки транзакций:
npx tsx src/index.ts rescue base --claim claims/мой-клейм.json --dry-run

# Если всё выглядит правильно — запускай:
npx tsx src/index.ts rescue base --claim claims/мой-клейм.json
```
